# debt-ops Star-Growth Plan

A plain-language, step-by-step guide to getting debt-ops as many GitHub stars
as possible over the next 90 days.

## How to use this folder

- **`README.md`** (this file) — the map. Start here.
- **`00-decisions-first.md`** — 3 calls you must make before anything else.
- **`01-foundation.md`** — Days 0–14. Fix the repo before any outreach.
- **`02-launch.md`** — Days 14–21. The one big launch moment.
- **`03-amplify.md`** — Days 21–60. Spread it everywhere.
- **`04-compound.md`** — Days 60–90. Make it keep paying off.
- **`research-notes.md`** — the raw research behind every step, with sources.

Each step is written so you can either **do it yourself** or hand it to an
agent. Steps that are worth researching deeper have a **"Hand to an agent"**
box with a ready-to-paste prompt.

## The 6 findings everything is built on

1. **Get into Anthropic's official plugin marketplace.** It ships with every
   Claude Code install. Biggest single lever.
2. **Sell the *artifact*, not the *mechanism*.** People star the debt registry
   they can screenshot — not "PostToolUse hooks."
3. **Position as "catches debt *as the agent writes it*."** That's the one
   thing audit tools and PR-review tools can't claim.
4. **Say it works across Claude Code, Cursor, and Codex** (if you'll commit to
   it). Winners all use multi-tool framing.
5. **Launch on Hacker News with numbers, not a demo.** Lead with the GitClear
   stats, not "I built a thing."
6. **The GitClear data is your hook.** Refactoring fell from ~25% to under 10%;
   copy-pasted code up 48%. Use these numbers everywhere.

## The shortest version

Fix the README → submit to Anthropic's marketplace → launch on HN with the
GitClear numbers → push it through newsletters/X/Reddit → publish a data essay
that keeps earning stars after the launch fades.
