write-article now mandates a verbatim repo masthead as the first body element of every article (applied to all 7 existing pieces). This deliberately overrides the skill's own research-backed ad-test principle — don't show the product before the reader has felt the problem, since one slop-smelling reader costs more than ten converting. Tradeoff taken on user instruction: warm README readers want the repo link in line one. Alternatives not taken: leave the masthead as unlinked text, or show it only on warm-slot (week-1/BOFU) pieces while cold data pieces stay link-free. Revisit if per-channel engagement shows cold-slot readers bouncing on the up-front link.
